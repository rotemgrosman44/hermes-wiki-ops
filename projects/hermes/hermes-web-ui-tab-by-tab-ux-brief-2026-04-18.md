# Hermes Web UI: Tab-by-Tab UX Brief

## Purpose
Use `Surfaces` as the canonical entry screen, then tighten the rest of the dashboard so each tab has one operational job and one human-gate style.

## Status
- Keep it as the health board.
- Add Hebrew-first operational headings and shorter helper copy.
- Surface `Needs Rotem` only for real interventions, not for passive metadata.
- Emphasize gateway health, fallback state, and blocked providers before secondary stats.

## Sessions
- Treat it as the live activity lane.
- Add clearer Hebrew scan labels for runtime, lane, and freshness.
- Keep aggregation visible, but show scope in one place only.
- Consider a simple urgency strip for sessions that actually need review.

## Skills
- Treat it as a controlled memory/tools screen, not a discovery wall.
- Add Hebrew helper copy explaining what is bundled, local, and external.
- Tighten density and reduce repetitive explanatory text.
- Surface drift-risk or modified skills more clearly than generic enabled/disabled state.

## Env
- Treat it as a sensitive configuration screen.
- Use Hebrew-first warnings and make source path/context immediately obvious.
- Distinguish safe read-only checks from risky edits.
- Keep long values compact with copy/reveal patterns like `Surfaces`.

## Cron
- Treat it as the automation lane.
- Prioritize what is running, what failed, and what needs Rotem.
- Surface the last run, next run, and failure reason in a compact operational strip.
- Consider a human-gate marker only for broken or approval-sensitive jobs.

## Logs
- Treat it as the investigation lane.
- Keep filtering and search primary, not decorative framing.
- Add Hebrew guidance for how to read the page: recent issues, blocked flows, and service-specific checks.
- Promote likely-actionable logs above raw volume.

## Dashboards To Consider Later
- `WhatsApp Operations`: onboarding, pairing, bridge health, repair.
- `Operator Oversight`: Telegram operator status, approvals, gates, posting readiness.
- `Human in the Loop`: one screen for decisions, approvals, content review, and intervention points.
- `Recursive Loop`: reports, cron outcomes, scorecards, and learning artifacts only.
